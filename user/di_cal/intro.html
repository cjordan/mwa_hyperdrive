mwa_hyperdrive documentation

DI calibration

Direction-Independent (DI) calibration "corrects" raw telescope data. hyperdrive achieves this with "sky model calibration". This can work very well, but relies on two key assumptions:

  • The sky model is an accurate reflection of the input data; and
  • The input data is not too contaminated (e.g. by radio-frequency interference).

A high-level overview of the steps to hyperdrive DI calibration is below. Solid lines indicate actions that always happen, dashed lines are optional:

The flowchart is made with the following code. Paste it into https://mermaid.live, edit, and insert the "kroki" link above.

%%{init: {'theme':'dark', 'themeVariables': {'fontsize': 20}}}%%
flowchart TD
    InputData[fa:fa-file-archive Input data files]-->Args
    SkyModel[fa:fa-file-alt Sky-model source-list file]-->Args
    Settings[fa:fa-cog Other settings]-.->Args

    Args[fa:fa-cog User arguments]-->Valid{fa:fa-code Valid?}
    Valid --> cal

    subgraph cal[For all timesteps]
        direction RL
        Read[fa:fa-code Read input data]
        Model["fa:fa-code Generate model vis\n (CPU or GPU)"]
        Model-.->WriteModelVis[fa:fa-save Write model visibilities]
    end

    cal--->LSQ[fa:fa-code Calibrate via least squares]
    LSQ-->|Iterate|LSQ

    LSQ-->Sols[fa:fa-screwdriver Calibration solutions]
    Sols-->WriteSols[fa:fa-save Write calibration solutions]
    Sols-.->CalVis[fa:fa-code Calibrate input data]
    CalVis-.->WriteVis[fa:fa-save Write calibrated visibilities]
    CalVis-.->WriteSubVis[fa:fa-save Write subtracted visibilities]
    Model-.->WriteSubVis
